<span data-ttu-id="f3020-153">Exchange 2013 ermöglicht datenbankmobilität um Postfach Zuverlässigkeit und Verfügbarkeit zu verbessern.</span><span class="sxs-lookup"><span data-stu-id="f3020-153">Exchange 2013 allows for database mobility to improve mailbox reliability and availability.</span></span> <span data-ttu-id="f3020-154">Datenbankmobilität können Sie Kopien der Exchange Store-Datenbank, trennen Sie die Datenbank vom Server und auf einem anderen Server zu speichern.</span><span class="sxs-lookup"><span data-stu-id="f3020-154">Database mobility enables you to make copies of your Exchange store database, disconnect the database from the server, and store it on another server.</span></span> <span data-ttu-id="f3020-155">In einer Exchange 2013 Database Availability Group (DAG), werden mehrere Kopien der Datenbank auf unterschiedlichen Computern gespeichert.</span><span class="sxs-lookup"><span data-stu-id="f3020-155">In an Exchange 2013 Database Availability Group (DAG), multiple copies of the database are stored on different computers.</span></span> <span data-ttu-id="f3020-156">Wenn eine oder mehrere Kopien der Datenbank von Hardware- oder andere Systemfehler verloren gegangen sind, kann Informationen von den anderen Kopien auf die Datenbank über normale Replikationsvorgänge Ausführen eines erneuten Seedings verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="f3020-156">When one or more copies of the database are lost due to hardware or other system failure, information from the other copies can be used to reseed the database via normal replication operations.</span></span>
  
<span data-ttu-id="f3020-157">Für backup-Vorgänge Wenn die Konfiguration der zu sichernden Exchange 2013-Datenbanken in einer DAG kann die backup-Anwendung besser Interferenzen zwischen der Momentaufnahme und der aktiven Server Performance verhindern, indem Sie den Snapshot auf einem der inaktiv Datenbank kopiert.</span><span class="sxs-lookup"><span data-stu-id="f3020-157">For backup operations, if the Exchange 2013 databases that are to be backed up are configured in a DAG, the backup application can better prevent interference between the snapshot and the active server's performance by taking the snapshot on one of the inactive database copies.</span></span> <span data-ttu-id="f3020-158">Da in den meisten Fällen Datenbankkopien synchronisiert werden, kann die backup-Anwendung Momentaufnahmen aus verschiedenen Kopien der Datenbank, und es später von der Pfadkomponenten wiederherzustellen.</span><span class="sxs-lookup"><span data-stu-id="f3020-158">Because the database copies are for the most part synchronized, the backup application can take snapshots from different copies of the database, and later reconstruct it from the pieces.</span></span>

Da die Datenbankprotokolldateien der Kopien geringfügig voneinander abweichen können, kann das Wiederherstellen einzelner Datenbankkopien unter Verwendung unterschiedlicher Daten dazu führen, dass die Datenbank nicht mehr bereitgestellt werden kann.</span><span class="sxs-lookup"><span data-stu-id="f3020-p112">The only supported method of restoring DAG databases from backup data is to restore all copies of the database by using the same backup data.
