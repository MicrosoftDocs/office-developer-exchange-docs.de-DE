## <a name="exchange-database-mobility"></a><span data-ttu-id="904c8-151">Exchange-Datenbankmobilität</span><span class="sxs-lookup"><span data-stu-id="904c8-151">Exchange database mobility</span></span>
<span data-ttu-id="904c8-152"><a name="bk_exchangedatabasemobility"> </a></span><span class="sxs-lookup"><span data-stu-id="904c8-152"><a name="bk_exchangedatabasemobility"> </a></span></span>

<span data-ttu-id="904c8-153">Exchange 2013 ermöglicht datenbankmobilität, um die Zuverlässigkeit und Verfügbarkeit von Postfächern zu verbessern.</span><span class="sxs-lookup"><span data-stu-id="904c8-153">Exchange 2013 allows for database mobility to improve mailbox reliability and availability.</span></span> <span data-ttu-id="904c8-154">Mithilfe der Datenbankmobilität können Sie Kopien Ihrer Exchange-Speicherdatenbank erstellen, die Verbindung zwischen der Datenbank und dem Server trennen und sie auf einem anderen Server speichern.</span><span class="sxs-lookup"><span data-stu-id="904c8-154">Database mobility enables you to make copies of your Exchange store database, disconnect the database from the server, and store it on another server.</span></span> <span data-ttu-id="904c8-155">In einer Exchange 2013 Database Availability Group (DAG) werden mehrere Kopien der Datenbank auf unterschiedlichen Computern gespeichert.</span><span class="sxs-lookup"><span data-stu-id="904c8-155">In an Exchange 2013 Database Availability Group (DAG), multiple copies of the database are stored on different computers.</span></span> <span data-ttu-id="904c8-156">Wenn Kopien der Datenbank aufgrund von Hardware- oder anderen Systemausfällen verloren gehen, können die Informationen aus anderen Kopien der Datenbank über gewöhnliche Replikationsvorgänge Seeding für das erneute Seeding verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="904c8-156">When one or more copies of the database are lost due to hardware or other system failure, information from the other copies can be used to reseed the database via normal replication operations.</span></span>
  
<span data-ttu-id="904c8-157">Wenn die Exchange 2013 Datenbanken, die gesichert werden sollen, in einer DAG konfiguriert sind, kann die Sicherungsanwendung für Sicherungsvorgänge besser Interferenzen zwischen der Momentaufnahme und der Leistung des aktiven Servers verhindern, indem Sie die Momentaufnahme auf eine der inaktiven Datenbankkopien nimmt.</span><span class="sxs-lookup"><span data-stu-id="904c8-157">For backup operations, if the Exchange 2013 databases that are to be backed up are configured in a DAG, the backup application can better prevent interference between the snapshot and the active server's performance by taking the snapshot on one of the inactive database copies.</span></span> <span data-ttu-id="904c8-158">Da die Datenbankkopien in den meisten Fällen synchronisiert werden, kann die Sicherungsanwendung Momentaufnahmen der verschiedenen Datenbankkopien erstellen und die Datenbank später aus den Datenelementen wiederherstellen.</span><span class="sxs-lookup"><span data-stu-id="904c8-158">Because the database copies are for the most part synchronized, the backup application can take snapshots from different copies of the database, and later reconstruct it from the pieces.</span></span>
