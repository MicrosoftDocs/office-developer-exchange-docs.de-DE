# <a name="argument"></a>Argument

Das **Argument** -Element gibt die Argumente für die Aktion an. 
  
```xml
<Argument Value=""/>
```

 **ProtectionRuleArgumentType**
## <a name="attributes-and-elements"></a>Attribute und Elemente

In den folgenden Abschnitten werden Attribute, untergeordnete und übergeordnete Elemente erläutert.
  
### <a name="attributes"></a>Attribute

|**Attribut**|**Beschreibung**|
|:-----|:-----|
|**Value** <br/> |Ein nicht leeren Zeichenfolgenwert, der den Wert eines Arguments der Aktionsteil einer Schutzregel darstellt. Dieses Attribut ist erforderlich.  <br/> |
   
### <a name="child-elements"></a>Untergeordnete Elemente

Keine.
  
### <a name="parent-elements"></a>Übergeordnete Elemente

|**Element**|**Beschreibung**|
|:-----|:-----|
|[Aktion (ProtectionRuleActionType)](action-protectionruleactiontype.md) <br/> |Gibt an, welche Aktion ausgeführt werden muss, wenn der Bedingungsteil der Regel übereinstimmt.  <br/> |
   
## <a name="text-value"></a>Textwert

Keine.
  
## <a name="remarks"></a>Hinweise

Das Schema, das dieses Element beschreibt, befindet sich im virtuellen IIS-Verzeichnis, das Exchange-Webdienste hostet.
