# <a name="securityidentifier"></a>SecurityIdentifier

Das Element **SecurityIdentifier** stellt Security Descriptor Definition Language (SDDL) Form einer Sicherheits-ID ( [SID](sid.md)).
  
```xml
<SecurityIdentifier/>
```

 **string**
## <a name="attributes-and-elements"></a>Attribute und Elemente

In den folgenden Abschnitten werden Attribute, untergeordnete und übergeordnete Elemente erläutert.
  
### <a name="attributes"></a>Attribute

Keine.
  
### <a name="child-elements"></a>Untergeordnete Elemente

Keine.
  
### <a name="parent-elements"></a>Übergeordnete Elemente

|**Element**|**Beschreibung**|
|:-----|:-----|
|[GroupIdentifier](groupidentifier.md) <br/> |Stellt eine einzelne Sicherheits-ID und das Attribut für eine Active Directory-Gruppe, Objekt, das Konto ein Mitglied ist.  <br/> Es folgt der XPath-Ausdruck, der dieses Element:  <br/>  `/SerializedSecurityContext/GroupSids/GroupIdentifier[i]` <br/> |
|[RestrictedGroupIdentifier](restrictedgroupidentifier.md) <br/> |Stellt die Gruppe Sicherheits-ID und die Attribute für eine eingeschränkte Gruppe innerhalb einer Benutzertoken.  <br/> |
   
## <a name="remarks"></a>Hinweise

Dieses Element wird in der Kopfzeile (SOAP = Simple Object Access Protocol) verwendet.
  
Das Schema, das dieses Element beschreibt, befindet sich im virtuellen EWS-Verzeichnis des Computers, der Microsoft Exchange Server 2007 mit installierter Clientzugriff-Serverrolle ausführt.
